# Tech Stack

## 全体方針
- 無料枠で継続運用できること
- 個人利用でも現実的な構成であること
- 実務に近い設計判断が伝わること

を前提に技術選定を行っています。

---

## Notion
- 日報管理にすでに使用しているツール
- UI/UX が整っており、入力コストが低い
- API によりデータ取得・更新が可能
- 状態管理（チェックボックス）との相性が良い

---

## Gemini API
- 無料枠があり、定期バッチ用途に適している
- 軽量モデルでも要約・フィードバック用途には十分
- API経由で明示的にプロンプト制御が可能

---

## AWS Lambda
- サーバレスで運用コストが低い
- バッチ処理との相性が良い
- 実行時間・回数が無料枠内に収まる

---

## AWS EventBridge
- 定期実行（平日夜間）を簡単に設定可能
- Lambda との統合がシンプル
- Cron管理をコードに持たなくてよい

---

## AWS SSM Parameter Store
- APIキーやDB IDを安全に管理
- コード内にシークレットを持たない設計
- Lambdaからの参照が容易

---

## CloudWatch Logs
- バッチ実行ログの確認
- 失敗時の調査
- 個人情報を含まない最小限ログ設計
